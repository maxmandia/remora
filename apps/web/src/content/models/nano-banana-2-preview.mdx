## The preview configuration behind Nano Banana 2

Nano Banana 2 Preview refers to `gemini-3.1-flash-image-preview`, the preview model that preceded Google’s stable `gemini-3.1-flash-image` release. It combined image generation and conversational editing with text, image, and PDF inputs. The model could return both image and text output and supported thinking during composition.

The preview introduced output choices at 0.5K, 1K, 2K, and 4K, with 1K as the default. It also expanded extreme aspect-ratio support with 1:4, 4:1, 1:8, and 8:1 formats. Search grounding could incorporate both text and image search results to inform a generation with current web context.

## Image controls and multimodal context

A prompt could generate a new image or revise visual input through a conversational sequence. Thinking could remain enabled or be turned off, and search grounding worked in either state. Google describes the family as optimized for high-volume, efficient image work compared with the more production-oriented Nano Banana Pro.

The underlying Gemini 3.1 Flash Image specification provides a 131,072-token input limit and a 32,768-token output limit. Batch processing and image generation are supported. Structured outputs, Live API use, URL context, file search, and audio generation are not part of this image model’s feature set.

## A historical exact variant

Google released the generally available `gemini-3.1-flash-image` model on May 28, 2026 and deprecated the preview identifier, with shutdown scheduled for June 25, 2026. This page remains separate because the supplied selection names the preview configuration rather than the later stable alias.

The lifecycle note matters operationally: new integrations should not target the retired preview ID. The page documents what the selected variant represented—its resolutions, search grounding, aspect ratios, thinking, and conversational editing—without claiming that the preview endpoint remains available. Generated images use Google’s provenance systems, including SynthID.
