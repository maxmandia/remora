## Standard Grok image generation

`grok-imagine-image` is xAI’s standard model for generating an image from a text prompt and editing an existing image with a new instruction. Its declared modalities are text and image input with image output. The generation endpoint accepts the model identifier, prompt, resolution, aspect ratio, output format, and optional batch count.

Both 1K and 2K resolutions are documented. Aspect-ratio control allows the canvas to be selected independently of the prompt, while a prompt describes subject matter, composition, lighting, style, and any text that should appear. Several images can be requested from one prompt when a workflow needs alternatives rather than a single result.

## Image-led edits

An edit request adds a source image. The prompt can direct a local change, restyle the entire composition, or replace visible elements while using the original as context. The standard model accepts an image URL or supported encoded input through xAI’s image API. A useful edit instruction identifies what to preserve as carefully as what to change.

The default response contains a temporary hosted URL. Base64 response data is available for applications that need the image bytes immediately. Files API storage can preserve an output beyond the temporary generation URL and can optionally expose a stable public link.

## Standard versus Quality

The standard model is separate from `grok-imagine-image-quality`. Both currently list 1K and 2K output, but they have different model identifiers and service profiles. This page therefore avoids treating “quality” as a request flag on the standard model.

xAI’s model listing also exposes the dated standard alias `grok-imagine-image-2026-03-02`. The API applies content-policy review to generated and edited images. Clients should handle failures using the returned API error and store successful results promptly when the temporary URL is not sufficient for their workflow.
