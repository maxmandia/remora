## An efficiency-focused Nano Banana model

Nano Banana 2 Lite is the product name for `gemini-3.1-flash-lite-image`. Google positions it as the efficiency specialist in its image generation family, designed for high-volume and interactive applications where latency and compute cost matter. It accepts text and image inputs and can return both images and text, supporting generation as well as conversational image editing.

The model generates at 1K resolution. Google documents 14 supported aspect ratios, including standard square, portrait, landscape, and ultrawide formats. The list includes 1:1, 3:2, 2:3, 3:4, 4:3, 4:5, 5:4, 9:16, 16:9, and 21:9. It does not support 2K or 4K output, which distinguishes Lite from the broader resolution range of Nano Banana 2 and Nano Banana Pro.

## Generation, editing, and thinking

Text-to-image requests can describe the desired scene directly. Image-plus-text requests support edits that retain visual context from supplied material. The model also supports function calling and two thinking levels, minimum and high. Google Search grounding is not available on the Lite image model, so prompts should include the factual or visual context the generation needs.

Generated images carry SynthID and C2PA provenance information. Batch processing is supported for workloads that do not need an immediate interactive response.

## Choosing Lite deliberately

The Lite variant is not simply a low-resolution switch on Nano Banana 2. It has its own stable model code, smaller output limit, supported feature set, and efficiency goal. It is appropriate when 1K output is sufficient and response time or generation volume is more important than 2K or 4K delivery.

Google lists a 65,536-token input limit and a 4,096-token output limit. Audio generation, Live API use, search grounding, structured outputs, and URL context are not supported. Those boundaries should be considered when a workflow combines image creation with broader multimodal reasoning.
