## Prompt-led video and sound

Wan 2.7 Text to Video turns a written brief into an MP4 clip with synchronized audio. Alibaba Cloud lists multi-shot narrative support for the `wan2.7-t2v` model ID, so the prompt can describe a progression across more than one camera setup rather than limiting every generation to a single unbroken view. Shot order, transitions, recurring subjects, and audio continuity should be stated clearly when using that capability.

The model supports 720p and 1080p output at 30 frames per second. Duration ranges from two to fifteen seconds. A two-second result can focus on one concise visual event, while a longer clip leaves room for multiple beats. The requested timeline still needs to remain realistic: more shots reduce the time available for each action to read clearly.

## Synchronized audio

Audio is part of Wan 2.7’s documented text-to-video feature set. Prompts can include dialogue, environmental sound, effects, and music cues that correspond to on-screen events. Alibaba also documents a dated June 12 checkpoint when direct custom-audio input is required. This page instead represents the unversioned `wan2.7-t2v` alias, which can advance as Alibaba updates the served family.

Because the model starts without an image, all persistent visual characteristics come from text. A useful prompt establishes character appearance, location, lighting, lens or framing, motion, and sound without changing those definitions midway through the clip.

## Alias versus checkpoint

The generic model ID and the dated `wan2.7-t2v-2026-06-12` checkpoint are separate selectable configurations. The alias is convenient for accessing the current Wan 2.7 text model, while a dated ID is clearer when a production workflow must pin the vendor revision. Both share the official 720p/1080p, two-to-fifteen-second, 30 fps specification at the time of this review. Image-conditioned generation uses a different Wan 2.7 model ID and adds first-frame, last-frame, continuation, and audio-driven inputs.
